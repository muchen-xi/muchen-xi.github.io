# 智能家居实验台设想

> ESP32 节点 + MQTT + 树莓派中枢

---

## 系统架构

```
ESP32 节点 × N  ──MQTT──▶  树莓派中枢  ──Web──▶  手机/电脑仪表盘
    │                           │
    ├─ BME280 温湿度            ├─ InfluxDB 时序存储
    ├─ BH1750 光照              ├─ Node-RED 自动化规则
    └─ 继电器控制               └─ Grafana 可视化面板
```

## 为什么选 MQTT？

- **轻量**：ESP32 轻松跑，一个 `PubSubClient` 库搞定
- **解耦**：传感器只负责发，中枢只负责收，互不依赖
- **标准**：IoT 事实标准，以后接入 Home Assistant 零门槛

## 第一步：单节点验证

ESP32 + BME280 → 每 30 秒上报温湿度 → MQTT broker（树莓派上跑 Mosquitto）→ 终端订阅查看。

## 第二步：多节点组网

3 个节点分布在不同房间 → 对比微气候差异 → 发现"北屋比南屋低 3°C"这种有趣的数据。

## 第三步：自动化

- 湿度 > 70% → 继电器开抽湿机
- 光照 < 100 lux + 时间 18:00-23:00 → 开灯
- 温度 > 28°C → 发手机通知

---

> 先让一个节点稳定跑一周，再加第二个。贪多嚼不烂。
